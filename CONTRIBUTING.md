## 1. Types of Contributions

We welcome:
- Bug fixes
- Performance improvements
- New PDF features (read, write, merge, split, extract text, metadata, etc.)
- Documentation improvements
- Code refactoring
- Usage examples

---

## 2. Contributions We Do Not Accept

Please do NOT:
- Use GitHub Issues for general Python support questions
- Submit untested or broken code
- Add large features without prior discussion in an Issue

---

## 3. Ground Rules

- Be respectful and professional
- Follow PEP 8 coding standards
- Write clean, readable, and maintainable Python code
- Add comments for complex logic
- Test your changes before submitting

---

## 4. First-Time Contributors

If you are new:
- Check issues labeled `good first issue` or `help wanted`
- Small fixes and documentation updates are great starting points
